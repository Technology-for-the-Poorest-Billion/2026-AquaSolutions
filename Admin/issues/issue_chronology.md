## Overview
The _issues_ folder provides the identified challenges for each stage of the project. These problems range from data quality concerns (key to the Week 1/2 pivot) to forecasted implementation complexities which would need to be addressed down the line. 

## v1
v1 refers to the issues we identified and predicted while examining the publicly available data for training a TinyML classifier. The most concerning are genuine feasibility difficulties due to the data shortcomings. 

## v2
v2 highlights many of the same data-based issues from v1 (section A), now confirmed after having run "ML/ML Full dataset/full_dataset_V1.ipynb" and "ML/ML Full dataset/full_dataset_V2.ipynb". Some new issues appear as well. Ultimately, the challenges identified in v2 formed the motivation for "Week 2"'s work, where we focused on a system for collecting appropriate data for our original task. 

## v3
v3 is a live document outlining the challenges related to the current idea's deployment in the field. It covers cultural issues, incentive structures, future ML considerations, and much more. It is to be updated as more work is done.
